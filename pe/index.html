model 04 — tax-benefit microsimulation · PolicyEngine

The economy, one household at a time.

PolicyEngine.py is the micro member of the suite: a tax-benefit microsimulation package for the UK and US that computes a specific family's taxes, benefits, and net income under current law or a reform — the same engine that powers policyengine.org. Where the macro members trace a reform through the whole economy, this one answers at person and household resolution.

Quickstart Read the code
01 — what it is

Statute as code, at household resolution.

The package encodes the UK and US tax-and-benefit rules as parameterised statute and computes every variable for a household you describe — for the UK: income tax, National Insurance, Universal Credit, Child Benefit, Pension Credit; for the US: federal and state income tax, payroll tax, SNAP, TANF, EITC, CTC, SSI, and Social Security. A reform is a flat dict of parameter path → new value — the same shape at household and population level; re-run the same family under the new law and read the difference.

The stack has four layers: policyengine-core (the simulation framework, originally forked from OpenFisca), policyengine-uk and policyengine-us (the statutory rules per country), and policyengine — the analyst-facing package this page covers, which adds dataset management, reforms, structured outputs, and version-pinned release bundles for reproducibility.

Within PolicyEngine Macro it is the complement to the macro engines. The OLG model already consumes PolicyEngine's effective and marginal rates to build its tax functions; this page is the direct route to the engine itself — for "what does this reform do to this family?" rather than "what does it do to GDP?".

02 — install

One package, two countries.

Published on PyPI as policyengine. The base install is the shared analysis tooling only; add the [models] extra to pull the pinned UK + US country models, or a country extra for just one. 

pip install policyengine            # shared tooling, no country models
pip install "policyengine[models]"  # + pinned UK and US models
pip install "policyengine[uk]"      # UK model only
pip install "policyengine[us]"      # US model only
03 — quickstart

A household in one call.

Describe the people, pick a year, and read the results. Pass a reform dict to run the same household under changed law. 

import policyengine as pe

# UK: single adult earning £50,000 — current law
uk = pe.uk.calculate_household(
    people=[{"age": 35, "employment_income": 50_000}],
    year=2026,
)
print(uk.person[0].income_tax)                   # £7,486
print(uk.person[0].national_insurance)           # £2,994
print(uk.household.hbai_household_net_income)    # £39,520

# US: single filer in California, with a reform applied
us = pe.us.calculate_household(
    people=[{"age": 35, "employment_income": 60_000}],
    tax_unit={"filing_status": "SINGLE"},
    household={"state_code": "CA"},
    year=2026,
    reform={"gov.irs.credits.ctc.amount.adult_dependent": 1000},
)
print(us.tax_unit.income_tax, us.household.household_net_income)

run the same household with and without the reform argument and difference the results — that is the household-level reform impact. For the £50,000 UK earner above, raising the basic rate from 20% to 25% ({"gov.hmrc.income_tax.rates.uk[0].rate": 0.25}) adds £1,872 of income tax — 5pp on ~£37.4k of basic-rate income — and cuts net income by the same £1,872.

04 — beyond one household

Population analysis on representative microdata.

The same package runs whole representative datasets — the UK Family Resources Survey (enhanced), the US CPS — through the model, so you can aggregate any variable by decile, entity, or demographic: 

import policyengine as pe
from policyengine.core import Simulation
from policyengine.outputs.aggregate import Aggregate, AggregateType

datasets = pe.uk.ensure_datasets(datasets=["enhanced_frs_2023_24"],
                                 years=[2026], data_folder="./data")
simulation = Simulation(dataset=datasets["enhanced_frs_2023_24_2026"],
                        tax_benefit_model_version=pe.uk.model)
simulation.run()

agg = Aggregate(simulation=simulation, variable="universal_credit",
                aggregate_type=AggregateType.SUM, entity="benunit")
agg.run()
print(f"Total UC spending: £{agg.result / 1e9:.1f}bn")

For baseline-vs-reform comparisons at population scale, the package bundles canonical workflows — pe.uk.economic_impact_analysis and its US counterpart — plus structured output types (Aggregate, ChangeAggregate, IntraDecileImpact) for decile changes, program statistics, poverty, inequality, and regional impacts. Each release pins the country-model and dataset versions it certifies, so a published estimate can be traced to the exact bundle that produced it.

UK population microdata lives in a private Hugging Face repository — set HUGGING_FACE_TOKEN from an account with access before running UK population examples. Household calculations need no data access at all.
05 — outputs & scope

What it answers — and what it doesn't yet.

Outputs are per-entity variables: incomes, each tax, each benefit, and net income at person, benefit-unit/tax-unit, and household level — plus aggregates and distributional statistics over microdata. Honest scope notes:

  • It is a static microsimulation at heart — reforms are modelled statically, with optional post-hoc behavioural responses (e.g. labour supply elasticities). No general equilibrium: that second act is exactly what the macro members of the suite add.
  • PolicyEngine Macro integration status: the household tools — computing one specific family's taxes, benefits, and net income under current law or a reform — are hosted and live on the MCP server (and the pe-macro CLI): calculate_household, household_reform_impact, and list_reform_parameters. Representative-population reform scoring (population_reform_impact: budgetary impact, deciles, winners/losers over whole microdata) is hosted too — the private enhanced-FRS credential is provisioned server-side — and runs locally with a HUGGING_FACE_TOKEN.
  • Two countries: UK and US.
