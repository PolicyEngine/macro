model 04 — tax-benefit microsimulation · PolicyEngine

The economy, one household at a time.

PolicyEngine.py is the micro member of the suite: a tax-benefit microsimulation package for the UK and US that computes a specific family's taxes, benefits, and net income under current law or a reform — the same engine that powers policyengine.org. Where the macro members trace a reform through the whole economy, this one answers at person and household resolution.

Quickstart Read the code
01 — what it is

Statute as code, at household resolution.

The package encodes the UK and US tax-and-benefit rules as parameterised statute and computes every variable — income tax, National Insurance, Universal Credit, SNAP, EITC, CTC, and the rest — for a household you describe. A reform is a parameter path and a new value, the same reform objects the rest of the suite scores; re-run the same family under the new law and read the difference.

Within MacroMod it is the complement to the macro engines. The OLG model already consumes PolicyEngine's effective and marginal rates to build its tax functions; this page is the direct route to the engine itself — for "what does this reform do to this family?" rather than "what does it do to GDP?".

02 — install

One package, two countries.

Published on PyPI. The default install pulls both country models; use an extra to install just one. 

pip install policyengine          # UK + US
pip install policyengine[uk]      # UK model only
pip install policyengine[us]      # US model only
03 — quickstart

A household in one call.

Describe the people, pick a year, and read the results. Pass a reform dict to run the same household under changed law. 

import policyengine as pe

# UK: single adult earning £50,000 — current law
uk = pe.uk.calculate_household(
    people=[{"age": 35, "employment_income": 50_000}],
    year=2026,
)
print(uk.person[0].income_tax)                   # income tax
print(uk.household.hbai_household_net_income)    # net income

# US: single filer in California, with a reform applied
us = pe.us.calculate_household(
    people=[{"age": 35, "employment_income": 60_000}],
    tax_unit={"filing_status": "SINGLE"},
    household={"state_code": "CA"},
    year=2026,
    reform={"gov.irs.credits.ctc.amount.adult_dependent": 1000},
)
print(us.tax_unit.income_tax, us.household.household_net_income)

run the same household with and without the reform argument and difference the results — that is the household-level reform impact.

04 — beyond one household

Population analysis on representative microdata.

The same package runs whole representative datasets — the UK Family Resources Survey (enhanced), the US CPS — through the model, so you can aggregate any variable by decile, entity, or demographic: 

import policyengine as pe
from policyengine.core import Simulation
from policyengine.outputs.aggregate import Aggregate, AggregateType

datasets = pe.uk.ensure_datasets(datasets=["enhanced_frs_2023_24"],
                                 years=[2026], data_folder="./data")
simulation = Simulation(dataset=datasets["enhanced_frs_2023_24_2026"],
                        tax_benefit_model_version=pe.uk.model)
simulation.run()

agg = Aggregate(simulation=simulation, variable="universal_credit",
                aggregate_type=AggregateType.SUM, entity="benunit")
agg.run()
print(f"Total UC spending: £{agg.result / 1e9:.1f}bn")
UK population microdata lives in a private Hugging Face repository — set HUGGING_FACE_TOKEN from an account with access before running UK population examples. Household calculations need no data access at all.
05 — outputs & scope

What it answers — and what it doesn't yet.

Outputs are per-entity variables: incomes, each tax, each benefit, and net income at person, benefit-unit/tax-unit, and household level — plus aggregates and distributional statistics over microdata. Honest scope notes:

  • It is a static microsimulation — no behavioural response, no general equilibrium. That second act is exactly what the macro members of the suite add.
  • Population-level reform scoring through the MacroMod integration layer is planned; today the hosted MCP server exposes the household calculator and household-level reform impacts.
  • Two countries: UK and US.
