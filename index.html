Open models for
the economy.

PolicyEngine Macro is a suite of five open, reproducible economic models: an OBR emulator and a Bank of England structural VAR for the UK, the Fed's FRB/US model in Python for the US, and the PSL overlapping-generations model and PolicyEngine microsimulation for both countries. Score reforms, simulate shocks, forecast GDP and inflation, and study long-run structural change — each model answers a different class of question, not one number scored five ways.

01 — the idea

From static scores to microfounded models.

Policy analysis usually stops at a static score — who pays what the morning after a reform. But people respond: work, saving, and investment shift; wages, prices, and interest rates move, and the fiscal arithmetic moves with them. Modeling that response has been the preserve of institutions with in-house model teams.

PolicyEngine Macro opens it up — the full economy's answer, a model away: an overlapping-generations engine for the long run, Python implementations of the OBR's and the Fed's official equation systems for the near term, and a Bayesian structural VAR that reads the economy in shock terms. Every engine open source, every result reproducible.

02 — the models

One suite, many lenses on the economy.

Each model answers a different class of question — near-term fiscal scenarios, forecasts and shock identification, long-run structural change, or household-level tax and benefit effects. PolicyEngine Macro gives them a common way to be discovered, run, and interpreted, while preserving each model's own assumptions and outputs.

psl-ogUKUSlocal only

PSL overlapping generations model

A dynamic general-equilibrium model of 80 age cohorts working, saving, and retiring — built on OG-Core, with tax functions estimated from PolicyEngine microdata. Solves the long-run steady state and the year-by-year transition path for GDP, investment, consumption, revenue, and debt.

obr-macroUKhosted

OBR macroeconometric model

A Python emulator of the OBR's own published forecasting model — 372 structural equations solved simultaneously by Gauss–Seidel iteration. Runs fiscal-multiplier and tax-shock experiments quarter by quarter, anchored to the official Economic and Fiscal Outlook.

read the model →
boe-svarUKhosted

Bank of England structural VAR model

A Python replication of the Bank of England's structural VAR for the UK — eight quarterly variables, six identified shocks. Decomposes GDP and inflation into their drivers, forecasts with credible bands, and explains forecast revisions.

read the model →
pe-microsimUKUShosted

PolicyEngine tax-benefit microsimulation

The micro member: PolicyEngine's UK and US tax-benefit microsimulation — the same engine behind policyengine.org. Computes a specific family's taxes, benefits, and net income under current law or a reform; population-level reform scoring runs hosted and locally.

read the model →
frb-usUSlocal

FRB/US

A from-scratch Python implementation of the Fed's large-scale US macroeconometric model, built from the public model definition and April 2026 LONGBASE data. VAR-expectations solver validated against the Fed's pyfrbus to ~1e-9; model-consistent expectations not yet implemented.

03 — the workflow

Different questions need different models.

Start with a question and choose the lens: the OBR emulator for near-term UK fiscal scenarios, the structural VAR for forecasts and what's driving the data, FRB/US for US macro dynamics, the overlapping-generations model for long-run structural change, and PolicyEngine for household and distributional effects. Each runs at its own resolution, on its own inputs — no single reform object flows through all five.

Every analysis records its assumptions, data vintage, model version, and units, and each result is a deviation from an official baseline in real-world units: £ per year for a household, £bn and % of GDP for the economy. One CLI, one MCP server, and one Python API reach every model.

Built in the open. Read it, run it, contribute.

Every model, dataset decision, and validation report lives in a public repository — nothing you can't rerun. File an issue, suggest the next model, or open a pull request.
