\section{Method and implementation}
\label{sec:implementation}

\subsection{Code architecture}

The replication is a small, dependency-light Python package (\texttt{boe\_var}, src layout; NumPy, SciPy, pandas, matplotlib) with a module per stage of the pipeline:

\begin{itemize}
\item \texttt{data.py} --- \texttt{load\_data()} returns the transformed quarterly panel (a \texttt{PeriodIndex} DataFrame in the Table~\ref{tab:variables} ordering, $100\cdot\log$ except Bank Rate), assembled by a separate download script from public sources;
\item \texttt{bvar.py} --- the \texttt{BVAR} class: NIW posterior sampling of $(\Pi, \Sigma)$ with Minnesota and sum-of-coefficients priors and exogenous Covid dummies, plus companion-form and residual helpers;
\item \texttt{identification.py} --- the Table~\ref{tab:restrictions} restrictions as machine-readable constants (\texttt{ZERO\_RESTRICTIONS}, \texttt{SIGN\_RESTRICTIONS}), the \citet{arias2018} \texttt{draw\_Q} routine, and \texttt{identify()}, which pairs posterior draws with accepted rotations;
\item \texttt{analysis.py} --- IRFs, FEVDs, estimated shocks, historical decompositions, forecast bands and the composite (forecast-revision) impulse response, with plotting helpers.
\end{itemize}

An end-to-end script reproduces the paper's Figures 2--6 and writes a numerical summary; the test suite (Section~\ref{sec:validation}) runs the same code on the real dataset with a fixed seed.

\subsection{Estimation and identification runs}

The production configuration draws \textbf{10{,}000} posterior draws, of which \textbf{751} pass the sign-restriction accept/reject step (a 7.5 per cent acceptance rate), with $p=4$ lags and hyperparameters $\lambda = 0.2$, $\mu = 1$. Zero restrictions are satisfied by construction on every draw; sign restrictions are enforced by rejection, so they too hold exactly on every accepted draw. Because the \citet{arias2018} zero-restriction sampler does not draw $Q$ uniformly from the restricted set, the slow validation path additionally computes the algorithm's importance weights (each requiring a numerical Jacobian per accepted draw); the importance-weighted effective sample size is \textbf{350.3}, and weighted and unweighted results are compared in Section~\ref{sec:validation}. The forecast-revision exercise re-estimates through 2024Q2 with 6{,}000 draws (487 accepted, weighted ESS 207.5).

\subsection{Hosted MCP tools}

Within PolicyEngine Macro the model is wrapped by an integration layer --- a \texttt{pe-macro} command-line interface and a Model Context Protocol (MCP) server, deployed automatically to Modal on every merge --- exposing three tools:

\begin{itemize}
\item \texttt{forecast\_uk} --- unconditional forecasts of the eight variables from the data edge, with median paths and credible bands, in year-on-year units for growth variables;
\item \texttt{latest\_shocks} --- the posterior sign probabilities and magnitudes of the six identified structural shocks in the most recent quarter, i.e.\ a structural reading of ``what just hit the economy'';
\item \texttt{model\_summary} --- the model card: specification, sample, restriction matrix, acceptance statistics and validation status.
\end{itemize}

\noindent The same tools back the model page at \url{https://macromod.vercel.app/svar/}. This makes the SVAR usable as the suite's conditioning member: agents and scripts can pull a structural baseline before invoking the reform-scoring models.
